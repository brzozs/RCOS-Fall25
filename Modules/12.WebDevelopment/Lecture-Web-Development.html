Open Source Tools

ReactJS

  • Most Popular JavaScript Library
  • Manged by Facebook
  • Known for its virtual Document Object Model (DOM)
  • Pros
    • Great use for high traffic
    • Allows for the creation of custom components
    • DOM model makes for Fast Rendering
  • Cons
    • Constantly being developed and changed
    • Poorly Documented due to the constant changes
  • React Homepage

NodeJs

  • Cross platform JavaScript Runtime environment
  • Used to create server side web applications
  • Developed in 2009
  • Pros
    • Built on Google Chrome’s V8 Engine meaning very fast execution time and run time
    • Node Package Manager is extremely large
    • Totally Asynchronous
    • Very easy to learn
  • Cons
    • Performance is reduced with heavy computational tasks
    • Many Libraries are poorly documented
    • Unstable API
  • Node Homepage

Express.js

  • Express.js is the most popular web framework for Node.js
  • Designed for building web applications and APIs
  • The "de facto" standard server framework for Node.js
  • Pros
    • Very Minimal and Lightweight
    • Middleware and database support
    • Support for databases such as mongodb and mysql
    • Scalability
  • Cons
    • No strict structure or convention
    • Limited built in features
    • Overhead
  • Express Homepage

AngularJS

  • AngularJS is a structural framework for dynamic web apps
  • Angular is essential what HTML would have been if it was designed for applications
  • Developed in 2010 and Maintained by Google
  • Uses Model View Controller (MVC) Architecture
  • Pros
    • Single Page Architecture
    • Design Architecture promotes consistency and scalability in mobile app development
    • Cross platform compatible
  • Cons
    • Limited SEO capabilities
    • Not suitable for smaller application
    • Slightly worse performance then other popular frameworks
  • Angular Homepage

Vue.js

  • Vue is a frontend JavaScript framework used to build user interfaces
  • Focuses only on the view layer
  • Released in 2014
  • Goal was to create a “lightweight Angular”
  • Pros
    • Easy to integrate with other libraries and existing projects
    • Very Lightweight
    • Single-file components and readability
  • Cons
    • Small community size
    • Not as many resources as other frameworks
  • Vue Homepage

Ember.js

  • Ember is a JavaScript Framework that makes building complex web application easier
  • Tools such as Ember Inspector, Ember CLI, and Ember Data that help developers create scalable and maintainable applications
  • Pros
    • Ember CLI allows for a streamlined development process
    • CoC (Convention over Configuration) design paradigm
    • Offers fast rebuild and auto reloads
  • Cons
    • Steep learning curve
    • Large function and class name make management of code base challenging
    • Complexity may not be best for smaller scale projects
  • Ember Homepage

Bootstrap

  • Bootstrap is the world’s most popular front-end web development framework
  • Based on HTML5, CSS, and JavaScript
  • Pros
    • Websites are very responsive
    • Simplifies tedious task such as resizing images and elements
    • Responsive community
    • User designed templates
  • Cons
    • Can become a crutch
    • Steep Learning curve
  • Bootstrap Homepage

Tailwind CSS

  • First CSS framework that provides a set of low-level CSS utility classes
  • Provides building blocks without having to write custom css
  • Pros
    • Allows for creation of complex elements very quick
    • Allows for consistent designs across a project
    • Gives a sweet spot between pure CSS and Bootstrap
  • Cons
    • Increases size of css files
    • Not easy to switch once you start with tailwind
  • Tailwind Homepage

Bulma

  • Bulma is a lightweight CSS framework that emphasizes simplicity and responsiveness
  • Known for its intuitive and beginner-friendly documentation
  • Pros
    • Easy to learn
    • Offers built in responsiveness
    • Designs have customizable options
  • Cons
    • Limited Component Library
    • Community and support not as strong as other frameworks
  • Bulma Homepage

jQuery

  • jQuery is a minified JS library created to simplify javascript operations
  • Pros
    • Cross Browser support
    • Core features such as event handling simplifies code for events
    • AJAX Methods
  • Cons
    • Does not allow backward compatibility
    • Massive library size
  • jQuery Homepage

MongoDB

  • MongoDB is a NoSQL document database
  • Used for high volume data storage
  • Uses both a document-oriented data model and an unstructured query language
  • Pros
    • Performs 100 times faster than other relational databases and provides high performance
    • Using simple query syntax
    • Easy environment and quick setup
    • MongoDB uses “sharding”, which expands the storage capacity. Allows MongoDB to use horizontal scalability
  • Cons
    • Uses ACID transactions which could lead to data corruption
    • Joins in MongoDB can be very tedious
    • High Memory Usage
  • MongoDB Homepage

Django

  • Django is a full-stack framework for web developers working with Python
  • Aim is to reduce the overall web development process when building web apps
  • Pros
    • Builds and scales up projects incredibly quickly
    • Django is well documented
    • Very Secure
  • Cons
    • Not great for smaller projects
    • Can slow down websites
  • Django Homepage

Figma

  • Figma is a cloud-based design tool that allows you to work together in real time
  • Google docs of UI design
  • Pros
    • Real time collaboration
    • Quick and easy file sharing
    • Vast library of plugins
  • Cons
    • Need to be online to use
    • Larger products must rely on paid plan
    • Need A Decent RAM & Graphic Cards To Run Smoothly
  • Figma Homepage

Gitlab

  • GitLab is a cloud-based Git repository and DevOps platform that makes
         it liquid-smooth for developers to test, monitor, and deploy code
  • Pros
    • Self-hosting capabilities
    • Supported CI/CD lifecycle
    • Easy code maintenance
  • Cons
    • Comparatively slower then Github
    • Devoid of some of the enterprise-level software development features
  • GitLab Homepage

GitHub

  • NOT OPEN SOURCE
  • Largest source code worldwide
  • Git-based repository hosting platform
  • Pros
    • Promotes Open source development
    • Quick and advanced search in the repository framework
    • Swift integration of third-party providers
  • Cons
    • Not Open source itself
    • Lack of a wide range of development functionalities
    • Has space limitations
  • GitHub Homepage

Web Development Process

Step 1: Planning

  • Consider Questions:
    • What is the goal of your website?
    • Who is your audience, and what do you want them to do on your website?
    • What type of website are you building?
    • What content are you aiming to publish, and at what volume?
    • What’s the purpose of this content?
    • How will you structure your website for the best navigational experience?
  • Create a sitemap
  • Create a wireframe
  • Choose a tech stack

Step 2: Designing

  • Begin working on the UI design of your webpage
  • Should choose color schemes, logo, images and videos to be used, etc.
  • Should be expanding on the wireframe

Step 3: Developing

  • Build the website!

Step 4: Testing

  • Website must be thoroughly tested before you publish it
  • Every link and function of the web page should be tested multiple times

Step 5: Deployment

  • Launch your brand new website!

Step 6: Maintenance

  • Once the website is launched the work is not done
  • Must continue to:
    • Implement new features
    • Fix bugs
    • Modify existing features
